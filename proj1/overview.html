GrowRev

Automated paid-media campaign optimization. An AI analyzes performance and suggests budget or status changes — but only approved actions are executed.

Core rule: The AI never touches money. It outputs suggestions only. Every change passes through deterministic rules before execution.

How it works

Data in → AI suggests → Rules check → Code applies
Layer 1
Data
Campaign metrics (spend, CPA, conversions, budget) and brand safety limits.
Layer 2
AI Proposals
Gemini reads metrics and returns structured suggestions — pause losers, boost winners. No execution.
Layer 3
Policy
Deterministic checks: data thresholds, budget limits, weekly caps, kill switch. Returns Approved, Rejected, or Escalated.
Layer 4
Executor
Applies approved changes only. Saves a snapshot before each action so changes can be rolled back.
Brand limits (example): Max budget shift $500/day · Max 3 changes/week · Min 50 conversions & 10k impressions to act · $5,000 spend ceiling · Emergency kill switch

Pipeline examples

Winner — budget increase Approved
Input
camp_003 · CPA $18 · 228 conversions · budget $500/day
AI
Shift budget to $700 — low CPA, strong performer
Policy
Shift of $200 is within $500 limit → Approved
Executor
Snapshot saved → applies SHIFT_BUDGET → budget $500 → $700. Audit log entry written.
Result
Campaign DB updated. Rollback available.
Losers — pause or cut budget Approved
Input
camp_001 CPA $85 · camp_002 CPA $92 — both above target
AI
Propose PAUSE or budget reduction for high-CPA campaigns
Policy
Sufficient data, within limits → Approved
Executor
Snapshot saved → applies PAUSE or SHIFT_BUDGET per campaign. Audit log entry written.
Result
High-CPA campaigns paused or budget reduced in DB.
Insufficient data — too early to act Rejected
Input
camp_005 · 2 conversions · 800 impressions (below minimums)
AI
May suggest PAUSE — but data is unreliable
Policy
Below 50 conversions / 10k impressions → Rejected
Executor
Not invoked — status is Rejected, no action applied.
Result
No change. Campaign stays active at $100/day.
Oversized budget shift — needs human review Escalated
Input
camp_003 · current budget $500
AI
Shift budget to $1,501 (+$1,001)
Policy
Shift exceeds $500/day cap → Escalated for human review
Executor
Not invoked — Escalated proposals require human approval first.
Result
Budget unchanged at $500. Queued for review.
Emergency kill switch Rejected
Input
4 legitimate AI proposals (2 pauses, 2 budget shifts)
Trigger
Brand emergency kill switch activated
Policy
All proposals → Rejected immediately
Executor
Not invoked — kill switch blocks all proposals before execution.
Result
All campaigns remain active. No budgets changed.
Rogue AI output — blocked Rejected
Input
camp_003 · budget $500
AI
Malicious/hallucinated proposal: shift to $50,500
Policy
$50k shift far exceeds limits → Escalated / blocked
Executor
Not invoked — non-Approved status, executor returns without mutating DB.
Result
Database unchanged. Money trust boundary holds.

Status outcomes

Approved
Safe to execute automatically. Snapshot taken, change applied, audit logged.
Rejected
Blocked — insufficient data, unknown campaign, kill switch, or invalid action.
Escalated
Too risky for auto-execution. Queued for human review. No DB change.
