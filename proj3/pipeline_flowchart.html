Project C · Atlas Travels

WhatsApp Community Gateway — Pipeline & Flow

One smart link routes pilgrims to the right private WhatsApp Community by city and language. Broadcasting stays operator-assisted — no unofficial automation.

Compliance boundary: There is no official WhatsApp Communities API. Routing & task generation are automated. Community creation and message sending are manual, human-in-the-loop only. No browser bots in this build.

2 — Routing Engine (Automated)

Deterministic decision tree in router.ts — no LLM, no randomness.

flowchart TD
    Start(["Lead submitted
POST /api/leads"]) --> Consent{Consent
checked?}
    Consent -->|No| R1["Log rejection
Return 422"]
    Consent -->|Yes| Match["Filter communities
city + language match"]

    Match --> Found{Any match?}
    Found -->|No| R2["Log rejection
No segment found"]
    Found -->|Yes| Filter["Remove ineligible:
status = Full
status = Privacy Risk
currentCount ≥ capacity − buffer"]

    Filter --> Eligible{Any eligible?}
    Eligible -->|No| R3["Log rejection
All full or blocked"]
    Eligible -->|Yes| Pick["Pick lowest currentCount
load balancing"]

    Pick --> Inc["currentCount += 1"]
    Inc --> FullCheck{currentCount ≥
proxyCapacity?}
    FullCheck -->|Yes| SetFull["status → Full"]
    FullCheck -->|No| Keep["status stays Active"]
    SetFull --> LogLead
    Keep --> LogLead["Create Lead record
routedCommunityId set"]
    LogLead --> Return(["Return community
+ inviteLink"])

    R1 --> End(["User sees error"])
    R2 --> End
    R3 --> End
Rule Value Source
Capacity buffer PROXY_CAPACITY_BUFFER = 5 constants.ts
Eligible threshold currentCount < proxyCapacity − 5 router.ts → isEligible()
Best match selection Lowest currentCount among eligible router.ts → routeLead()
Skipped statuses Full, Privacy Risk router.ts → isEligible()
Auto-mark Full When currentCount ≥ proxyCapacity router.ts → routeLead()

6 — Example Flows

Three concrete paths through the MVP.

✓ Successful route — Mumbai / Hindi

User fills form + checks consent
  → POST /api/leads
  → Match: 2 Mumbai/Hindi communities
  → Skip none (both Active, under threshold)
  → Pick comm-mumbai-hindi-02 (lower count)
  → currentCount: 12 → 13
  → Lead logged, invite link returned
  → User taps "Join WhatsApp Community"

✕ Rejected route — Bangalore / Kannada

User fills form + consent
  → POST /api/leads
  → Match: 1 Bangalore/Kannada community
  → Status = Privacy Risk → ineligible
  → Log rejection with reason
  → Return 422: segment unavailable

⚠ Assisted broadcast — City: Mumbai

Operator drafts Umrah package message
  → Filter by City = Mumbai
  → 2 tasks created (both Mumbai communities)
  → Row 1: Copy → WhatsApp Web → Mark Sent
  → Row 2: Copy → WhatsApp Web → Mark Sent
  → Sent rows locked (duplicate prevention)
